# API Service Design

## Procedure

1. Define API boundaries and contracts.
2. Specify request/response schemas and validation rules.
3. Design error model and status code strategy.
4. Define authn/authz and sensitive-data handling.
5. Add health/readiness endpoints.
6. Add observability (structured logs, metrics, traces if available).
7. Define load and resilience expectations.

## Output format

- Endpoint map
- Schema and validation plan
- Error handling policy
- Security controls
- Operational checklist
- Test strategy
