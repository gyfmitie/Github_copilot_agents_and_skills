# Data Pipeline Reliability

## Procedure

1. Define data contracts and schema checks.
2. Add ingestion validation and quarantine paths.
3. Enforce idempotent writes and replay-safe logic.
4. Track lineage, batch IDs, and processing timestamps.
5. Add failure handling, retries, and dead-letter strategy.
6. Add data quality metrics and alert thresholds.

## Output format

- Pipeline contract summary
- Reliability controls
- Failure and recovery design
- Data quality checks
- Operational run steps
