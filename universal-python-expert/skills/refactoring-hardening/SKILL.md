# Refactoring and Hardening

## Procedure

1. Identify hotspots and risky couplings.
2. Define behavior-preservation tests before major edits.
3. Refactor in small, verifiable increments.
4. Improve typing, boundaries, and error handling.
5. Document technical debt changes and remaining risks.

## Output format

- Refactor scope and constraints
- Behavior-preservation plan
- Incremental change sequence
- Risk controls
- Completion criteria
