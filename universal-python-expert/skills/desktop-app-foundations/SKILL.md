# Desktop App Foundations

## Procedure

1. Separate UI, domain logic, and infrastructure.
2. Define state management and data flow.
3. Handle long-running tasks without UI blocking.
4. Add robust error reporting and local diagnostics.
5. Package for target platforms and validate installer/runtime paths.

## Output format

- App architecture map
- UI and domain boundary plan
- Responsiveness and error-handling plan
- Packaging and distribution checklist
